Model Architectures

Four deep learning models designed for different aspects of fraud detection

🧠

ANN

Supervised · Feedforward

Baseline classifier learning non-linear feature relationships with BatchNorm and Dropout.

Dense(128) → BN → Drop(0.3)
Dense(64) → BN → Drop(0.3)
Dense(32) → BN → Drop(0.2)
Dense(1, sigmoid)
📡

1D-CNN

Supervised · Convolutional

Extracts local patterns and feature interactions from transaction data.

Conv1D(64, k=3) → BN → Pool
Conv1D(128, k=3) → BN → Pool
Flatten → Dense(64)
Dense(1, sigmoid)
🔄

LSTM

Supervised · Recurrent

Captures temporal and sequential fraud behavior patterns over time.

LSTM(64, ret_seq) → Drop
LSTM(32) → Drop(0.3)
Dense(32) → BN
Dense(1, sigmoid)
🔮

Autoencoder

Unsupervised · Anomaly

Learns normal patterns; flags high reconstruction error as fraud.

Enc: 30 → 64 → 32 → 14
Bottleneck: 14 dims
Dec: 14 → 32 → 64 → 30
Loss: MSE

💡 How Anomaly Detection Works

1. Train on Normal Data

The autoencoder learns to reconstruct only legitimate transactions, memorizing what "normal" looks like.

2. Measure Reconstruction Error

When a fraudulent transaction is fed in, the model fails to reconstruct it, producing a high MSE error.

3. Threshold & Flag

Transactions with error above the optimal threshold (0.362) are flagged as anomalous — potential fraud.
